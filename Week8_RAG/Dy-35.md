# 1. Introduction to Retrieval Systems

Large Language Models (LLMs) are powerful, but they have limitations:

* Knowledge may be outdated
* Cannot access private company data
* Can generate hallucinations (incorrect information)

To solve these issues, modern AI applications use **Retrieval-Augmented Generation (RAG)**.

RAG allows AI systems to retrieve relevant information from external sources before generating a response.

# 2. Embeddings

## What is an Embedding?

An embedding is a numerical representation of text that captures its semantic meaning.

Humans understand:

```text
Dog
Cat
Tiger
Lion
```

Computers understand:

```text
[0.23, 0.81, -0.44, 0.67, ...]
```

Embeddings convert text into vectors so machines can process meaning mathematically.

---

## Example

Text:

```text
Machine Learning
```

Embedding:

```text
[0.12, 0.45, -0.67, 0.89, ...]
```

---

## Why Are Embeddings Important?

Computers cannot understand language directly.

Before searching or comparing text:

```text
Text
 ↓
Embedding Model
 ↓
Vector
```

The embedding preserves meaning.

---

## Semantic Understanding

Sentences with similar meanings produce similar vectors.

Example:

```text
I love dogs.
```

```text
I like puppies.
```

Their embeddings will be very close.

---

## Visual Representation

```text
          Animals

             Lion
              ●

Dog ●                     ● Tiger

             ● Cat

---------------------------------

          Vehicles

Car ●

Truck ●
```

Similar concepts cluster together in vector space.

# 5. Similarity Search

## What Problem Does It Solve?

Suppose a user asks:

```text
How does AI help doctors?
```

Document contains:

```text
Artificial Intelligence assists healthcare professionals.
```

Keyword search may fail.

Similarity search succeeds because it compares meanings.

---

## Similarity Search Workflow

### Step 1

Convert documents into embeddings.

```text
Document
 ↓
Embedding
```

### Step 2

Convert user query into embedding.

```text
Query
 ↓
Embedding
```

### Step 3

Compare vectors.

### Step 4

Return the closest vectors.

---

## Goal

Find the vectors nearest to the query vector.

```text
Query Vector

      ●

      ● Document A

            ● Document B

                  ● Document C
```

Document A is most similar and gets retrieved.

---

# 6. Cosine Similarity

The most popular similarity metric.

Measures the angle between vectors.

## Formula

cos(θ) = (A · B) / (||A|| × ||B||)

---

## Interpretation

| Score | Meaning   |
| ----- | --------- |
| 1     | Identical |
| 0     | Unrelated |
| -1    | Opposite  |

---

## Example

```text
Dog ↔ Puppy
```

Similarity:

```text
0.95
```

Very similar.

---

```text
Dog ↔ Car
```

Similarity:

```text
0.10
```

Not similar.

# 7. Vector Indexing

## The Problem

Imagine:

```text
10 Million Documents
```

Without indexing:

```text
Query
 ↓
Compare Against Every Vector
```

This is computationally expensive.

---

## Solution: Indexing

Indexing creates a smart structure for faster retrieval.

Think of a textbook:

```text
Book
 ↓
Index Page
 ↓
Jump Directly to Topic
```

Instead of reading the entire book.

---

## Benefits

* Faster search
* Better scalability
* Reduced computation
* Lower latency

---

# 8. Types of Vector Indexing

## 1. Flat Index

### How It Works

Searches every vector.

### Pros

* Highest accuracy

### Cons

* Slow for large datasets

---

## 2. IVF (Inverted File Index)

### How It Works

Groups vectors into clusters.

```text
Query
 ↓
Relevant Cluster
 ↓
Search Within Cluster
```

### Pros

* Faster than Flat Index

### Cons

* Slight accuracy tradeoff

---

## 3. HNSW

### Full Form

Hierarchical Navigable Small World

### Why It's Popular

* Very fast
* High accuracy
* Excellent scalability

Used by many modern vector databases.

# 11. RAG (Retrieval-Augmented Generation)

## What is RAG?

RAG combines:

1. Retrieval
2. Generation

The retriever finds relevant information.

The LLM generates the final answer.

---

## RAG Pipeline

```text
Documents
     ↓
Chunking
     ↓
Embeddings
     ↓
Vector Database

--------------------------------

User Query
     ↓
Query Embedding
     ↓
Similarity Search
     ↓
Relevant Chunks
     ↓
LLM
     ↓
Final Answer
```

---
